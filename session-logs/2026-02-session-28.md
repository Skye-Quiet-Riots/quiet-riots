# Session 28 — Post-Signup Onboarding Wizard

**Date:** 2026-02-25
**Branch:** `claude/onboarding-wizard`
**PR:** #65 (merged)

## What Was Done

Built the post-signup onboarding wizard — a 3-step flow new users see after signing up:

1. **Interest selection** — Grid of 16 issue categories with emojis, user picks 1+
2. **Language picker** — All 45 supported locales with native names, auto-detects from browser
3. **Country selector** — All 249 countries from DB with search/filter, option to skip

### Database changes

- Migration 014: `user_interests` table (user_id + category composite PK) + `onboarding_completed` column on users
- Updated schema.ts with new table definition and drop order

### API routes

- `POST /GET /api/users/me/onboarding` — Save interests + language/country preferences, check completion status
- `GET /api/countries` — Returns all countries (aggressive caching: 1h client, 24h CDN)

### Frontend

- `/[locale]/onboard` page — Client component with step-by-step wizard, progress bar, auth check
- Signup callbackUrl changed from `/` to `/onboard` (both OAuth and email)
- Completed users redirected away from /onboard automatically

### Translations

- 15 Onboarding keys added to all 45 locale files (en.json + 44 translations)
- Used parallel sub-agents for translation efficiency

### Tests

- 6 tests for interest queries (save, replace, single, all 16, empty, sort order)
- 7 tests for onboarding API (save + mark complete, language/country, 401, validation)
- Total: 1134 tests (up from 1121)

## Decisions

- **Simple redirect approach over auth callback**: Changed callbackUrl in signup page rather than adding complex redirect logic to auth.ts callbacks — simpler, easier to understand
- **Atomic batch for interests**: DELETE all + INSERT new in single db.batch() — handles re-saves cleanly without upsert complexity
- **Country API with aggressive caching**: Country data rarely changes, so 1h client / 24h CDN cache is appropriate
- **Skip option for country**: Country is optional in onboarding — users can set it later in profile

## Surprising

- ESLint pre-commit hook caught 2 unused variable warnings that blocked the initial commit attempt (unused `session` destructuring and unused `NextResponse` import). Good that hooks catch these.
- Translation sub-agent hit a "prompt too long" error when given 21 locale files at once — had to split into smaller batches
