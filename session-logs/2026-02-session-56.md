## Key Decisions

1. **Two explicit `as const` arrays** for `ALL_LOCALES` and `NON_EN_LOCALES` (not derived via `.filter()`) — preserves TypeScript tuple type needed by `z.enum()`
2. **Do NOT spread `ALL_LOCALES`** — `[...ALL_LOCALES]` creates mutable `string[]` which destroys the `Locale` union type
3. **Graceful degradation strategy**: Error for `set_language` (user explicitly chose it), silent fallback for queries, silent strip for auth routes
4. **Model upgrade**: `claude-haiku-4-20250414` was returning 404 — updated to `claude-haiku-4-5-20251001`
5. **Native script post-processing**: AI translation models (both Haiku and Sonnet) leak native script chars into romanised output. A transliteration cleanup pass is needed after API translation for `-Latn` locales.
6. **Dropped stale Serbian (`sr`)**: Was only in seed-reference-data.ts, no message/translation files existed

## Discovered / Surprising

- **AI models struggle with Greeklish and Cyrillic translit**: Even Sonnet mixed in ~589 Greek chars and ~729 Bulgarian Cyrillic chars in first pass. Haiku was worse. Cross-script contamination also happens (Cyrillic chars in Arabic files, Greek chars in Russian files).
- **Haiku model ID changed**: `claude-haiku-4-20250414` no longer exists, must use `claude-haiku-4-5-20251001`
- **Translation models rename JSON keys**: `ta-Latn` and `te-Latn` had `Home.riotersFunny` renamed to `Home.riotersFound` — model translated the key name, not just the value
- **Spread of `as const` creates typed array**: `[...NON_EN_LOCALES]` creates `("es" | "fr" | ...)[]`, not `string[]`. Using `.includes(l)` where `l` is `string` causes TS2345. Fix: explicit `string[]` type annotation.
