## Key Decisions

1. **Phase order:** Executed ~~0~~→~~1+2~~→~~4~~→~~3~~→~~6~~→~~5~~ (all done). Next: 7→8.
2. **Org health weighted averaging:** Used rioter_count as weight for health metrics; falls back to simple average when all counts are 0.
3. **Translation namespace prefix:** When key names collide across namespaces, always use `Namespace.key` format in translation files passed to `apply-ui-translations.js`.
4. **MobileBottomNav search:** Opens NavSearch in full-screen mobile overlay mode rather than navigating to /issues.
5. **iOS safe area:** Used inline `style={{ paddingBottom: 'env(safe-area-inset-bottom, 0px)' }}` since Tailwind's `pb-[env(...)]` would require custom config.

## Surprising Things

- **React state batching in tests:** Within a single `act()` block, React batches state updates, so sequential events like ArrowDown then Enter don't flush intermediate state. Must use separate `act()` blocks for each.
- **jsdom drops `env()` CSS:** `env(safe-area-inset-bottom)` is silently dropped by jsdom — can't test inline styles that use it.
- **`apply-ui-translations.js` namespace resolution:** The script finds keys by searching en.json for the first namespace containing that key name. When names collide (e.g., `actionsTitle` in both IssueDetail and OrgDetail), it matches the wrong namespace. Workaround: always prefix with `Namespace.key`.
- **next/image `alt=""` removes `role="img"`:** Decorative images with empty alt text don't get the `img` ARIA role in testing-library, so `getAllByRole('img')` doesn't find them. Use `querySelector` instead.
