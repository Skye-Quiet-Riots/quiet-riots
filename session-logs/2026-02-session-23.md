## What was worked on

### 1. Global rearchitecture plan (6 phases)

Designed and refined a comprehensive plan to transform Quiet Riots from a UK-centric, English-only platform into a global social platform comparable to X/Instagram/Facebook. The plan covers:

- **Phase 0:** Database foundation (migrations + types + seed scripts) — implemented this session
- **Phase 1:** Auth.js v5 + world-class signup/login (Google, Facebook OAuth + email magic links)
- **Phase 2:** i18n infrastructure (next-intl, `[locale]` URL routing, 40+ languages, RTL support)
- **Phase 3:** Multi-currency wallets + country-scoped content
- **Phase 4:** Privacy, compliance & account safety (GDPR, cookie consent, account deletion, data export)
- **Phase 5:** Social platform UX polish (onboarding wizard, user blocking, content reporting)
- **Phase 6:** Bot multilingual support

Plan file: `.claude/plans/generic-petting-sloth.md`

### 2. Security audit (incorporated into plan)

- CSRF protection via Origin/Referer validation on all mutation routes
- Email normalization (lowercase) to prevent duplicate accounts
- OTP-verified account linking (replaced email-match approach — attack vector)
- Email enumeration prevention (generic responses on signin/signup)
- Session invalidation via `session_version` counter in JWT

### 3. Edge case audit (incorporated into plan)

- Wallet idempotency fix (pre-existing double-credit bug)
- Campaign contribution race condition (move status update into `db.batch()`)
- Zero-decimal currency handling (JPY, KRW)
- RTL layout support (Tailwind logical properties)
- ICU MessageFormat for complex plural rules (Arabic has 6 forms)
- Bot edge cases (deactivated users, phone recycling, per-user rate limiting)

## Decisions made

1. **Auth provider:** Auth.js v5 (free, self-hosted) with custom Turso adapter. Providers: Google + Facebook + email magic links (Resend). Apple deferred until Apple Developer account is set up.
2. **i18n routing:** `[locale]` URL prefix (`/en/issues`, `/fr/issues`) — SEO-friendly, industry standard
3. **Translation APIs:** DeepL (European/major) + Google Cloud Translation (wider coverage like Swahili, Tamil)
4. **Currency:** Full multi-currency wallets, currency locked once set (no auto-conversion on country change)
5. **Account linking:** OTP-verified only (not email-match) — prevents identity hijacking
6. **Session strategy:** JWT with `session_version` counter for invalidation
7. **All 249 countries from day one** — user in any country should see their country in the dropdown

## Anything discovered or surprising

- libSQL fully supports `ALTER TABLE ADD COLUMN` with defaults — no table-copy pattern needed
- Expanding the `User` TypeScript interface with optional fields didn't break any existing tests (651 tests still passed) because test DB rows get cast via `as unknown as User`
- The existing wallet has a pre-existing idempotency bug documented in `wallet.test.ts` — a double-completed topup credits the wallet twice. This needs fixing in Phase 3 before adding multi-currency.
