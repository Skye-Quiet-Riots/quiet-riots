### 3. Comprehensive security/i18n audit of magic-link-email feature

Ran a full audit across security, i18n, and test coverage. **No code fixes were made** — the session was cut short before implementation. Findings documented below for next session.

#### Security issues found (CRITICAL)

1. **No HTML escaping in email template** — `url` and all `strings.*` values are interpolated directly into HTML via template literals. An attacker-influenced `callbackUrl` containing `"` could break out of the `href` attribute. Translation strings with `<script>` tags would render unescaped. Need an `escapeHtml()` utility.

2. **No URL scheme validation** — `javascript:` protocol URLs would render as clickable links in the email. Need to reject any URL not starting with `https://`.

3. **No rate limiting on `/api/auth/[...nextauth]`** — POST handler (which triggers magic link emails) has no rate limiter. An attacker could exhaust the Resend API quota or flood a victim's inbox.

#### i18n issues found

4. **No RTL support in email template** — `<html>` element has no `dir` or `lang` attribute. Arabic, Hebrew, and Farsi users get LTR email layout.

5. **8 missing locale message files** — et (Estonian), lt (Lithuanian), lv (Latvian), mk (Macedonian), sq (Albanian), sr (Serbian), ur (Urdu), zh (Chinese Simplified) have no message file at all. The `zh-CN.json` file exists on disk but the expected locale code is `zh`.

6. **8 unexpected locale files exist** — ca (Catalan), eu (Basque), gl (Galician), ml (Malayalam), ta (Tamil), te (Telugu), tl (Tagalog), zh-CN — not in the expected 45-locale list.

#### Missing tests found

7. **`sendVerificationRequest` has ZERO tests** — the main exported function that calls the Resend API is completely untested. Need mocked `fetch` tests for happy path, Resend errors (400/401/429/500), network failures, and payload shape.

8. **No non-English locale loading test** — `loadEmailStrings` is only tested with English and an invalid locale. No test verifies that e.g. French returns French strings.

9. **No parameterised locale coverage test** — should verify all 45 locales have all 5 email keys.

10. **No XSS regression tests** — no test verifies that HTML-dangerous characters in URL or strings are escaped.

#### Existing test suite status

- 732 tests passing across 49 files
- 11 test files failing (all due to `next-intl/server` import resolution in Vitest — pre-existing issue, not related to this feature)

## Gotcha discovered

- **Env vars added after auto-deploy need a manual redeploy** — the post-merge checklist checks that env vars exist and runs a health check, but doesn't trigger a redeploy when env vars were added *after* the auto-deploy built. The health endpoint only checks DB connectivity, not whether Resend/email is configured. Added to CLAUDE.md.
