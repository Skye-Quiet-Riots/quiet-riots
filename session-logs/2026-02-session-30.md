## What Was Done

### Connected accounts management on profile page

Replaced the static read-only connected accounts display with an interactive `ConnectedAccounts` client component that lets users link and unlink Google/Facebook OAuth accounts.

#### API route: `GET/DELETE /api/users/me/accounts`

- **GET** — Returns all connected accounts for the logged-in user
- **DELETE** — Unlinks a connected account by provider name. Validates: auth, rate limit, Zod schema, supported provider list (google/facebook), and safety guard (cannot unlink your last account — returns 409)

#### Query functions added to `src/lib/queries/users.ts`

- `countUserAuthMethods(userId)` — counts linked accounts
- `unlinkUserAccount(userId, provider)` — removes account link from DB

#### Client component: `ConnectedAccounts`

- Shows Google and Facebook with branded SVG icons
- Connected providers show green "Connected" badge + "Unlink" button
- Unconnected providers show "Link" button (triggers `signIn()` from next-auth/react)
- Unlink button is disabled (with tooltip) when it's the user's only connected account
- Error display for failed operations

#### Profile page update

- Server component passes initial account data; client component handles interactions

## Decisions

- **Safety guard at API level, not just UI:** The API returns 409 if you try to unlink your last account, even though the UI also disables the button. Defense in depth.
- **Supported providers list in route:** Only `google` and `facebook` can be unlinked via the API. Email/resend accounts are not listed as unlinkable (they're managed differently — via email verification).
- **Using `signIn()` for linking:** The "Link" button for unconnected providers triggers Auth.js `signIn(provider)` which handles the OAuth flow and account linking via the adapter's `linkAccount()` method. No custom linking API needed.
