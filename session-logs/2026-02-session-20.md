### Key decisions

- **Dev fallback key preserved for local/test** — rather than removing the fallback entirely (which would break `npm test` without env setup), kept it but made it rejected in production via `IS_USING_DEV_KEY && process.env.NODE_ENV === 'production'` check.
- **Phone validation as `.refine()` not `.transform()`** — `normalizePhone()` throws on invalid input, which would cause unclear Zod errors. Used `.refine()` with explicit error message + `.transform(s => s.trim())` instead.
- **Session auth on user routes, not bot routes** — the bot route authenticates via Bearer token and resolves users by phone number. The web API routes (`/api/users/[id]`, `/api/suggestions`, `/api/users/[id]/met-assistants`) now use session cookies. No overlap — bot never calls these endpoints.
- **Auto-deploy after PR creation** — going forward, after creating a PR: check CI passes → check if production DB needs migration/seed → merge → confirm deployment.

### Discoveries

- **Zod `.transform()` after `.optional()` breaks** — the order matters: `.max(255).transform(fn).optional()` works, but `.optional().transform(fn)` throws because `undefined` gets passed to the transform function.
- **Bot tests used invalid E.164 phones** — `+0000000000` starts with `+0` which isn't valid E.164 (must start with `+[1-9]`). The new phone validation caught this.
- **`sanitizeText` was defined but never imported** — existed in `src/lib/sanitize.ts` since it was written but zero imports across the entire codebase. Same for `normalizePhone()`.
