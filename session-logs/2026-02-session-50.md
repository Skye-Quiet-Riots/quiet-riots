# Session 50 — Stripe-Compliant Payment Copy (2026-02-26)

## What Happened

Audited the entire codebase for Stripe compliance issues and reframed all payment-related copy from donation/crowdfunding language to service-purchase language. This is required because Stripe **prohibits** donations to non-registered entities and **restricts** crowdfunding platforms (needs pre-approval).

## Key Decisions

1. **Vocabulary rules established:**
   - "campaigns" → "action projects"
   - "backers" → "supporters"
   - "contribute/contribution" → "pay/support/payment"
   - "raised" → "committed"
   - "funded" → "goal reached"
   - "disbursed" → "completed"
   - "fund" → "commission" (for campaign descriptions)
   - Never say "donate", "donation", "crowdfund", "fundraise", "raise funds"

2. **Framing:** Users purchase collective action services (legal reviews, audits, testing), not make charitable donations. This keeps Quiet Riots firmly in Stripe's "allowed" category.

3. **Bot personality updated:** SOUL.md now has a permanent Stripe Compliance section so the bot never uses prohibited language in any conversation.
