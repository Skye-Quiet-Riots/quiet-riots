## What Was Done

### 1. Fixed English Text Leaking into Non-English Bot Responses (PR #132)

A Bengali-speaking WhatsApp user received bot responses where issue names were translated but assistant data (quotes, bios, goals, focus text) and menu labels remained in English. Two root causes:

1. **SKILL.md templates** were all in English — the bot copied template examples verbatim instead of translating menu choices/labels
2. **`get_category_assistants` and `get_assistant_detail` bot actions** didn't accept `language_code` — assistant data always returned in English

#### Implementation (6 phases):

- **Phase 1:** Added `translateCategoryAssistant()` and `translateCategoryAssistants()` to `src/lib/queries/translate.ts` — overlays 7 fields (agent_quote, human_quote, agent_bio, human_bio, goal, focus, focus_detail)
- **Phase 2:** Updated bot route (`src/app/api/bot/route.ts`) — added `language_code` param to both actions' Zod schemas
- **Phase 3:** Updated web API routes (`src/app/api/assistants/route.ts` and `[category]/route.ts`) — added `locale` query param support
- **Phase 4:** Added tests for both bot and web API endpoints
- **Phase 5:** Updated `scripts/seed-translations.ts` — added `category_assistants` section (16 categories × 7 fields = 112 strings per locale)
- **Phase 6:** Generated translations for all 44 locales and applied to staging + production DBs

### 2. Built API-Based Translation Pipeline (PR #133)

User asked why sub-agent translations always take so long (~20 min for 44 locales). Built `scripts/translate.ts` — a fast API-based pipeline using the Anthropic SDK.

#### How it works:

- Sends translation requests directly to Claude Haiku API
- 44 parallel requests (concurrency limit of 10) instead of 6 serial sub-agent batches
- ~1-2 minutes total vs ~20 minutes for sub-agents
- Validates JSON output, strips markdown code fences, checks key counts
- Supports `--section`, `--all`, `--locales`, `--dry-run`, `--model` flags

#### CLAUDE.md updates:

- Added `npm run translate` command
- Added "DB Entity Translation Protocol (MANDATORY)" section — **sub-agents must never be used for DB entity translations going forward**
- Updated i18n checklist to reference `npm run translate`

## Decisions Made

1. **API pipeline over sub-agents:** Sub-agents are slow (serial file reads, Edit tool splicing, ~20 min). The API pipeline sends parallel requests to Claude Haiku and gets structured JSON back (~1-2 min). Made mandatory in CLAUDE.md.
2. **Haiku as default model:** `claude-haiku-4-20250414` for cost-efficient translations. Can override with `--model` flag for higher quality.
3. **Concurrency limit of 10:** Avoids Anthropic API rate limits while still being fast.
4. **Kept sub-agent protocol for UI translations:** `messages/*.json` translations still use sub-agents since they need file editing (reading existing files, splicing in new keys). The API pipeline is for DB entity translations where the whole section is replaced.

## Gotchas Discovered

- **ANTHROPIC_API_KEY not in sandbox bash:** The key exists in Claude Code's environment but isn't forwarded to child bash processes. Script loads from `.env.local` as fallback.
- **tsx doesn't load .env.local:** The translate script manually parses `.env.local` since tsx doesn't auto-load it (same issue as other scripts).
