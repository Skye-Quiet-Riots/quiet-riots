## Decisions Made

1. **45 supported locales** — en, es, fr, de, pt, pt-BR, it, nl, sv, da, no, fi, pl, cs, sk, hu, ro, bg, hr, sl, uk, ru, tr, ar, he, fa, hi, bn, ta, te, ml, th, vi, id, ms, zh-CN, zh-TW, ja, ko, tl, sw, el, ca, eu, gl
2. **Server components use `getTranslations()` (async)** — requires `await` in tests with pattern: `const el = await Component({props}); render(el);`
3. **Client components use `useTranslations()` (sync)** — no test changes needed
4. **ICU plurals not parsed by test mock** — test assertions match raw ICU strings like `{count, plural, one {country} other {countries}}`
5. **Global Vitest setup file** — `src/test/setup-intl.ts` mocks `next-intl`, `next-intl/server`, and `@/i18n/navigation` using actual English strings from `messages/en.json`
6. **eslint-disable for pre-existing `<img>` tags** — error pages, global-error, YouTube thumbnails use `<img>` intentionally (external URLs or outside Next.js tree)

## Discovered / Surprising

- **Async server component testing pattern:** `render(<Component />)` returns nothing for async components. Must call the component as a function and await it: `const el = await Component({props}); render(el);`
- **ICU MessageFormat not parsed by simple mock:** The `createTranslator` mock only handles `{param}` substitution, not ICU plural blocks. Tests must assert against raw ICU strings.
- **Pre-commit hook catches pre-existing warnings:** Moving files to `[locale]` makes lint-staged re-lint them, surfacing old `@next/next/no-img-element` warnings that were previously not caught.
- **`global-error.tsx` can't use `next/image`:** It renders outside the Next.js app tree (renders its own `<html>` tag), so `next/image` optimization isn't available.
