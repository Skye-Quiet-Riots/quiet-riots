## What was done

### Email auth provider (Resend) ‚Äî full setup

1. **Custom `sendVerificationRequest`** (`src/lib/magic-link-email.ts`):
   - Extracts user's locale from the `callbackUrl` query param in the verification URL
   - Dynamically imports the correct locale message file for email strings
   - Renders a branded HTML email with Quiet Riots logo, styled CTA button, and translated text
   - Includes plain text fallback
   - Falls back to English if locale loading fails

2. **Root-level redirect pages** to fix Auth.js locale prefix mismatch:
   - Auth.js `pages` config uses `/auth/signin`, `/auth/verify`, `/auth/error` (no locale prefix)
   - Actual pages live at `/[locale]/auth/verify`, etc.
   - Created redirect pages at `src/app/auth/{signin,signup,verify,error}/page.tsx` and `src/app/onboard/page.tsx`
   - Error page preserves `?error=` query param

3. **Locale-aware callbackUrl** in signin/signup forms:
   - `signin-form.tsx`: callbackUrl = `/${locale}` (homepage in user's language)
   - `signup-form.tsx`: callbackUrl = `/${locale}/onboard` (onboarding in user's language)

4. **i18n email translations** ‚Äî 5 new keys in all 45 locales:
   - `emailSubject`, `emailHeading`, `emailBody`, `emailButton`, `emailIgnore`
   - Translated via 4 parallel sub-agents (11 locales each)

5. **Vercel env vars** added:
   - `RESEND_API_KEY` (production + preview)
   - `EMAIL_FROM` = `Quiet Riots <noreply@quietriots.com>` (production + preview)

6. **Wired into Auth.js** (`src/lib/auth.ts`):
   - Resend provider now uses custom `sendVerificationRequest`
   - `from` defaults to `EMAIL_FROM` env var

### Protocol updates

- Added to CLAUDE.md: "Design for multiple languages and countries from the outset" convention
- Added ESLint pre-commit check protocol to auto memory (always run `npm run lint` before committing)

## Decisions

- **Root-level redirects default to `/en/`** ‚Äî Auth.js doesn't pass locale context in its page redirects, so we redirect to English. The magic link callback itself respects the locale from the callbackUrl.
- **`redirect: false`** in signIn calls ‚Äî means the "check your email" state is shown inline, not via a server redirect. This avoids the locale mismatch issue for the happy path.
- **Brand name untranslated** ‚Äî "Quiet Riots" kept as-is in all locale email translations.

## Gotcha discovered

- **Auth.js `pages` config has no locale prefix** ‚Äî Auth.js redirects to paths like `/auth/verify` but our pages are at `/[locale]/auth/verify`. Without root-level redirect pages, users hit 404s. This affects error redirects and the `verifyRequest` page.
