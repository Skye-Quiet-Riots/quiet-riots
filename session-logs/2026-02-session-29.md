# Session 29 — Avatar Upload (Vercel Blob)

**Date:** 2026-02-25

## What Was Done

Implemented avatar upload with Vercel Blob storage, displayed on profile page and nav bar.

### API route

- `POST /api/users/me/avatar` — Accepts multipart/form-data, validates MIME type (JPEG/PNG/WebP only), enforces 2MB max, deletes old blob if exists, uploads to Vercel Blob with `addRandomSuffix: true`, updates user DB record
- `DELETE /api/users/me/avatar` — Deletes blob from Vercel storage, clears `avatar_url` in DB
- Rate limited, session cookie auth

### Frontend

- `AvatarUpload` component — clickable avatar circle with hover overlay ("Change"), hidden file input, remove button, error display, loading state
- Profile page uses `<AvatarUpload>` instead of static initial letter
- Nav bar conditionally shows `<Image>` (from `session.user.image`) or initial letter fallback

### Auth.js session callback

- Consolidated `session_version` check and `avatar_url` fetch into a single DB query
- Avatar URL attached to `session.user.image` so it's available client-side via `useSession()`

### Config

- `next.config.ts` — Added `images.remotePatterns` for Vercel Blob, Google, and Facebook avatar URLs

### Translations

- 3 keys (avatarUpload, avatarChange, avatarRemove) added to all 45 locale files
- Used parallel background agent for the 44 non-English translations

### Tests

- 11 tests: upload + DB update, 401 auth, unsupported file types, GIF rejection, file size limit, no file provided, correct blob path pattern, old blob deletion, blob upload failure, DELETE removes avatar, DELETE 401
- Total: 1145 tests (up from 1134)

## Decisions

- **`addRandomSuffix: true`** for Vercel Blob — avoids cache issues when user re-uploads same file type. Each upload gets a unique URL.
- **Delete old blob before uploading new** — prevents storage buildup. Non-fatal if deletion fails (old blob may already be gone).
- **Auth.js session callback for avatar** — fetches `avatar_url` from DB on every session check rather than storing in JWT. Changes reflect immediately without token refresh.
- **Single DB query** in session callback — consolidated `session_version` + `avatar_url` fetch to avoid extra round trip.
