## What Was Done

### Seed translations script (`scripts/seed-translations.ts`)

Created a two-phase script for translating all DB content (issue names, org names, category names) into 44 non-English locales:

- **`--generate` mode**: Creates `translations/` directory with `en.json` baseline + placeholder files for all 44 locales (214 strings each: 16 categories + 49 issues×2 + 50 orgs×2)
- **`--apply` mode**: Reads translation JSON files, looks up entity IDs from DB, batch-inserts into `translations` table using `db.batch(statements, 'write')` with `upsertTranslation()`
- Exports `CATEGORIES`, `ISSUES`, `ORGANISATIONS`, `TranslationFile` for test use
- Guard at bottom prevents `main()` from running on import (important for test isolation)

### Translation generation (12 parallel agents)

Used 12 concurrent sub-agents to translate all 44 locales simultaneously:

- Spanish, French, German/Portuguese/pt-BR, Italian/Dutch/Swedish/Danish, Norwegian/Finnish/Polish/Czech, Slovak/Hungarian/Romanian/Bulgarian, Croatian/Slovenian/Ukrainian/Russian, Turkish/Greek/Arabic/Hebrew/Persian, Hindi/Bengali/Tamil/Telugu/Malayalam, Chinese(CN+TW)/Japanese/Korean, Thai/Vietnamese/Indonesian/Malay, Tagalog/Swahili/Catalan/Basque/Galician
- Organisation names preserved in English across all locales (brand names)
- Total: 45 JSON files (en + 44 locales), ~14-16KB each

### Translation tests (`scripts/seed-translations.test.ts`)

223 new tests validating:

- All 45 translation files exist with correct structure
- 16 category keys present in every locale
- 49 issue keys with name+description in every locale
- 50 org keys with name+description in every locale
- No empty values
- Brand name preservation for org names

## Decisions

- **Two-phase workflow**: `--generate` creates static JSON files (no DB needed), `--apply` inserts into DB (needs DB access). Separation allows reviewing translations before applying.
- **Brand name preservation**: Organisation names (e.g., "Thames Water", "Network Rail") kept in English across all locales — only descriptions translated.
- **Guard pattern for scripts**: `if (process.argv[1]?.endsWith('seed-translations.ts'))` prevents `main()` from running when imported by tests.

## Discoveries

- Some languages translate "Transport" identically (Dutch, German) — can't use `value !== english` as a completion check
- `gh pr merge` fails in worktrees — use `gh api repos/.../pulls/N/merge -X PUT -f merge_method=squash` as fallback
