## Key Decisions

1. **`translateAny()` is safe by design** — only overlays fields that (a) exist on the entity, (b) are currently `string | null`, (c) have a non-empty translation value. Prevents rogue translation rows from corrupting numeric fields.
2. **Bot notification messages stay English** — internal admin notifications (Setup Guide alerts) are not translated; only user-facing bot responses use `getBotMessage()`
3. **`getBotMessage()` uses dynamic imports** — same pattern as `reset-password-email.ts`, with Node.js module cache preventing re-reads. Additional `messageCache` Map for the `BotMessages` namespace extraction.
4. **Validation warnings are expected** — the translate script warns about "missing keys" when the model translates issue name keys (which is correct behavior for `issue_per_riot`), and about "extra keys" for the same reason.
