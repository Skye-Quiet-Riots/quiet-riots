## What Was Done

Implemented all 6 Phase 0 foundational fixes from the website redesign plan (PLAN.md):

### 0a. Global `:focus-visible` styles
- Added to `src/app/globals.css`: `:focus-visible { outline: 2px solid var(--brand-blue); outline-offset: 2px; }`
- Enables keyboard users to navigate all interactive elements

### 0b. `formatRelativeTime()` with Intl.RelativeTimeFormat
- Created `formatRelativeTime(dateStr, locale)` in `src/lib/format.ts`
- Uses `Intl.RelativeTimeFormat` with `numeric: 'auto'` and `style: 'narrow'`
- Replaced hardcoded English `timeAgo()` function in `feed-post-card.tsx`
- 6 new tests covering English, French, Japanese locales

### 0c. SafeUserProfile projection
- Created `SafeUserProfile` type and `safeProfile()` function in `src/types/index.ts`
- Strips: `email`, `phone`, `password_hash`, `password_changed_at`, `session_version`, `merged_into_user_id`
- Applied to bot API `identify` action response
- 7 new tests for field exclusion + 2 bot PII exclusion tests

### 0d. `t.rich()` replacing `dangerouslySetInnerHTML`
- Homepage `basedOn` message now uses `t.rich('basedOn', { em: (chunks) => <em>{chunks}</em> })`
- All 55 non-English locale files already had `<em>` tags — no locale updates needed

### 0e. `viewport-fit=cover` in root layout
- Added `export const viewport: Viewport` to `src/app/layout.tsx`
- Enables `env(safe-area-inset-bottom)` for Phase 5's mobile bottom nav on iOS

### 0f. SectionNav ARIA tablist upgrade
- Added `role="tablist"`, `role="tab"`, `aria-selected`, `tabIndex` management
- Arrow key navigation (ArrowRight/Left/Up/Down, Home, End)
- Ref-based focus management via `useRef<Map<string, HTMLButtonElement>>`

## Test Fixes Required

- **Format tests:** `Intl.RelativeTimeFormat` narrow style produces "5m ago" not "5 minutes ago" — loosened regex patterns
- **Bot/integration tests:** Existing tests asserted `user.phone` which SafeUserProfile now strips — updated to `toBeUndefined()`

## Decisions

1. Used `style: 'narrow'` for `Intl.RelativeTimeFormat` to match the original compact display
2. SafeUserProfile uses destructuring pattern (not `Object.entries` filter) for type safety
3. Did NOT add viewport-fit to individual pages — root layout export covers all pages
