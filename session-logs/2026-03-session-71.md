## Key Decisions

1. **Feed is issue-scoped, not org-scoped:** The `feed` table has no `org_id` column, so the intersection page shows the full issue feed. This matches the existing `getFeedPosts(issueId)` pattern.
2. **Evidence is org-scoped:** `getEvidenceForIssue(issueId, orgId)` already accepts an optional `orgId` parameter, so the intersection page filters evidence to the specific org.
3. **Pivot table links are conditional:** The `issueId` prop is only passed when rendering from the issue detail page. The org detail page does NOT pass `issueId`, so org-mode pivot rows still link to issue detail pages (not intersection pages).
