## What Was Done

Renamed 8 i18n key names across all 45 locale files to remove Stripe-prohibited terminology from the codebase internals. This was a continuation of the session 52 Stripe compliance work — session 52 renamed bot actions and database columns, this session cleaned up the remaining non-compliant i18n key names.

### Approach

Used a Node.js script to batch-rename keys across all 44 non-English locale files in one pass — much faster and more reliable than the parallel sub-agent approach that timed out in the previous session attempt.

## Context

The previous session (52) attempted this same work but ran out of context window while the 6 parallel sub-agents were renaming locale files. All edits were lost because nothing was committed. This session restarted from scratch with a simpler approach (Node.js script instead of sub-agents) and committed incrementally.

## Decisions

- **No display text changes** — The user-visible translated values were already Stripe-compliant (e.g., "Support This Project", "spent", "Project payment"). Only the internal key names were non-compliant.
- **No database changes** — This was purely a code/i18n rename.
- **Node.js script > sub-agents** — For simple key renames, a script is faster and more reliable than launching 6 parallel translation agents.
