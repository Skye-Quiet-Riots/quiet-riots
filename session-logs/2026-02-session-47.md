## What Was Done

### 1. Fixed OTP Delivery Script API Key (Critical Bug)

**Problem:** During E2E testing, phone OTP codes were not being delivered via WhatsApp. The delivery script (`scripts/deliver-otp-codes.sh`) was reading `BOT_API_KEY` from `.env.local`, which has the dev fallback key (`qr-bot-dev-key-2026`). Production rejects the dev key, so the script's API calls were silently failing with auth errors.

**Root cause:** The script's fallback mechanism (`BOT_API_KEY="${BOT_API_KEY:-production-key}"`) only activates when the variable is empty/unset, but `.env.local` explicitly sets it to the dev key — so the fallback never triggered.

**Fix:** Hardcoded the production API key directly in the script, since it always talks to production Vercel. Removed the `.env.local` sourcing entirely.

**PR #109** — merged, production verified healthy.

## Gotchas Discovered

- **`.env.local` dev key overrides production fallback in scripts:** Any script that reads BOT_API_KEY from `.env.local` will get the dev key, which production rejects. Scripts that always talk to production must hardcode the production key.
