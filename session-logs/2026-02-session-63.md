# Session 63 — Auto-Translate Issues on Approval

**Date:** 2026-02-28
**Branch:** `claude/auto-translate-on-approval`
**PR:** #156

## What Was Done

### Problem
Issues created via the Setup Guide bot flow (e.g., "Mobile Data Charges", "Fly Tipping") had no translations. The `seed-translations.ts --generate` reads from hardcoded arrays, not the DB, so runtime-created issues were missed. Non-English users saw raw English text.

### Solution: Auto-translate on approval via `after()`

1. **Added `triggerAutoTranslation()`** (`src/lib/queries/generate-translations.ts`) — orchestrates generate→mark-ready flow
2. **Wired into bot approve handler** (`src/app/api/bot/route.ts`) — uses `after()` from `next/server` for guaranteed non-blocking execution on Vercel
3. **Updated approval notification** — tells First Rioter translations are being generated automatically
4. **Backfilled 2 missing issues** — added "Mobile Data Charges" and "Fly Tipping" to `scripts/seed-translations.ts` and regenerated all 56 locale files

### Tests added
- 3 unit tests for `triggerAutoTranslation()` (success, failure, org path)
- 1 integration test verifying `after()` is called on approval
- `vi.hoisted()` mock for `after` in bot-api.test.ts (required because `after()` throws outside Next.js request scope)

## Key Decisions

- **`after()` over `await`**: Guide gets instant response; Vercel guarantees `after()` completes even after response
- **`after()` over fire-and-forget**: Bare promises can be killed when Vercel terminates the function post-response
- **`.catch(() => {})` inside `after`**: Silent failure is acceptable — `goLiveSuggestion()` requires `translations_ready` status as a structural safety net
- **Extracted `triggerAutoTranslation()`**: Testable in isolation without mocking `after()`. Reusable if web approval is added later

## Gotchas Discovered

- **`after()` throws synchronously outside request scope**: Must be mocked in Vitest since tests call `POST(request)` directly without a Next.js server
- **`vi.mock` hoisting**: `vi.mock` factory is hoisted above variable declarations — use `vi.hoisted()` to define mock variables that the factory references
