# Session 57 — Fix Entity Name Translations + Bot Label Translation

**Date:** 2026-02-27
**Branch:** `claude/fix-entity-name-translations`
**PR:** #138 (merged)

## What Was Done

### Problem
The WhatsApp bot was showing English text to non-English users:
- Issue names ("Train Cancellations") untranslated in all 55 locale translation files
- Template labels ("AI Agent", "Human Organiser", "Trending") copied verbatim from SKILL.md
- Bot not using translated API responses for entity names

### Root Cause
The `translate.ts` prompt said "Translate ONLY the string values. Keep ALL JSON keys exactly as they are." The AI model saw `"Train Cancellations": { "name": "Train Cancellations", ... }` and didn't translate the `name` value because it looked like the key/reference.

### Fix
1. **Updated `translate.ts` prompt** — Added explicit rule: "The `name` field values inside issues and organisations ARE user-facing display names — you MUST translate them."
2. **Re-ran translation pipeline** for `issues` and `organisations` across all 55 locales (110 successful, 0 errors)
   - bn-Latn and hi-Latn required a Sonnet re-run (Haiku left Latin-script names in English)
3. **Updated SKILL.md** with stronger label translation instructions:
   - Added translation reference table (English → Banglish → Spanish → French)
   - Replaced key templates with `{translate "..."}` / `{translated name from API}` pattern
   - Added bold reminders at template sections
4. **Updated test** — `seed-translations.test.ts` now allows translated org names (previously expected all to stay English)
5. **Applied translations** to staging (23,419) and production (24,497)
6. **Cleared OpenClaw sessions** for SKILL.md changes

## Decisions Made

- **Org names ARE translatable** — Most org names are brand names (stay English), but generic ones like "Local Councils" should be translated. Test updated to check for non-empty strings rather than exact English match.
- **Sonnet for romanised locales** — Haiku treats Latin-script English names as "already romanised" and doesn't translate them. Sonnet correctly translates to Banglish/Hinglish. Future runs for `-Latn` locales may need `--model claude-sonnet-4-20250514`.
- **SKILL.md partial template update** — Updated the first few key templates with `{translate "..."}` syntax rather than all ~30 templates. The label reference table + bold reminders should be enough for the bot LLM to generalise.

## Surprising Discoveries

- **All 55 locales had untranslated issue names** — This wasn't a bn-Latn-specific bug. Every locale file had English names. The translation pipeline had never translated them since the project started.
- **Haiku vs Sonnet for romanised locales** — Haiku sees "Train Cancellations" in a Banglish context and thinks "that's already Latin script, no translation needed." Sonnet understands the intent and produces "Train Cancel" (Banglish) / "Train Cancel Ho Jaana" (Hinglish).
