## Decisions Made

1. **Feed route path:** `/api/users/me/feed` instead of `/api/feed/personal` — benefits from existing proxy.ts cache header exemption for `/me` paths
2. **ON DELETE CASCADE:** Added to user_follows migration (fix it at creation time, not later)
3. **Search rate limit:** 30/min (not 15 — too restrictive with 200ms debounce)
4. **No duplicate functions:** Phase 7 reuses existing getEvidenceForIssue + getFeedPosts instead of creating duplicates
5. **UNION ALL aliasing:** riot_reels.upvotes→likes, submitted_by→user_id via LEFT JOIN, status filter on reels
6. **Load more (not infinite scroll):** Keeps footer reachable, caps at 5 pages

## Surprising Findings

- `feed` table has ZERO indexes — never indexed since creation
- Homepage uses `dangerouslySetInnerHTML` (line 177) — needs t.rich() replacement
- Root layout has no `viewport` export at all — `env(safe-area-inset-bottom)` silently evaluates to 0
- `riot_reels` uses `submitted_by` (nullable) instead of `user_id` — asymmetric with feed/evidence tables
- evidence table column is `org_id` not `organisation_id` — inconsistent naming
