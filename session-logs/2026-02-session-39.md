## Decisions made

- **FirstRioterBadge i18n via labels prop**: Rather than making FirstRioterBadge a client component (just to use `useTranslations`), kept it as a sync server component and pass translated labels from the parent async server component via a `labels` prop. Cleaner separation.
- **Category names in suggestion form**: Use `useTranslations('Categories')` to translate category dropdown options, reusing the existing `Categories` i18n section rather than duplicating.
- **apiValidationError returns 400**: Discovered that `apiValidationError()` defaults to status 400 (not 422). Consistent with existing patterns in the codebase.
