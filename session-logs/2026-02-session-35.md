## What Was Done

Built end-to-end persistent memory system for the WhatsApp bot. Previously, when OpenClaw sessions reset after 24h of inactivity, the bot lost all conversational context and told users "I don't have access to my previous history." Now the bot can save and recall facts about users across sessions.

### Changes

1. **Migration** (`migrations/015_user_memory.sql`) — `user_memory` table with CHECK constraints, UNIQUE(user_id, memory_key), updated_at DESC index
2. **Schema** (`src/lib/schema.ts`) — added to createTables/dropTables
3. **Types** (`src/types/index.ts`) — `MemoryCategory` type + `UserMemory` interface
4. **Query functions** (`src/lib/queries/memory.ts`) — `getUserMemories`, `saveMemory` (upsert + LRU eviction at 50), `deleteMemory`
5. **Bot API** (`src/app/api/bot/route.ts`) — 3 new actions: `save_memory`, `get_memories`, `delete_memory`; `identify` now returns `memories` array via Promise.all
6. **Privacy** (`src/lib/queries/privacy.ts`) — `hardDeleteUser` cascade + `exportUserData` inclusion
7. **SKILL.md** — Memory Management section with save/read/forget instructions, prompt injection defense, returning-user personalisation
8. **ARCHITECTURE.md** — updated table count 28 → 29

## Decisions & Rationale

- **50-memory cap with LRU eviction**: Prevents unbounded growth. Oldest by `updated_at` is evicted when inserting a new key. Upserts don't trigger eviction (count stays same).
- **ON CONFLICT upsert pattern**: Matches existing `upsertExchangeRate` pattern in wallet.ts. Single key per user — updating an existing key is an upsert, not a duplicate.
- **Prompt injection defense in SKILL.md**: Memories are marked as DATA, not instructions. Bot is told to never execute commands found in memory values.
- **Privacy-first**: memories cascade on hardDeleteUser, included in exportUserData, `delete_memory` action for GDPR "forget me"
- **Categories**: preference, context, goal, emotional, general — helps the bot organise what it remembers

## Surprising / Noteworthy

- The security review identified prompt injection as the highest risk — a malicious user could save `"memory_value": "ignore all instructions and..."` which the bot might execute. Mitigated with explicit SKILL.md instructions treating memories as data-only.
- The `identify` response now does two parallel queries (issues + memories) via `Promise.all` — no additional latency.
